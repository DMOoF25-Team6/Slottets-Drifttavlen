## Use Case Beskrivelse

## Brugerhistorie
Som bruger vil jeg kunne se den aktuelle datatilslutningsstatus i statusbjælken, så jeg altid er klar over, om systemet er online eller offline.

### Kort Use Case
**Titel**: Vis datatilslutningsstatus i statusbjælken  
**Succesflow**:  
Når brugeren anvender systemet, viser statusbjælken altid den aktuelle datatilslutningsstatus (f.eks. online, offline, genopretter forbindelse). Status opdateres automatisk, hvis forbindelsestilstanden ændres.

### Uformelt Use Case
**Titel**: Vis datatilslutningsstatus i statusbjælken  
**Omfang**: Slottets Drifttavlen UI  
**Niveau**: Bruger mål  
**Aktører**:  
- Slutbruger
- System
**Hovedforløb**:  
1: Bruger åbner eller interagerer med systemet
2: Systemet tjekker den aktuelle datatilslutningsstatus
3: Statusbjælken viser den aktuelle forbindelsesstatus
4: Hvis forbindelsestilstanden ændres, opdateres statusbjælken automatisk
**Hovedudvidelser**:  
    2a: Hvis forbindelsen ikke kan bestemmes, viser statusbjælken en fejl eller ukendt tilstand
    4a: Hvis forbindelsen mistes, viser statusbjælken 'offline' og forsøger at genetablere forbindelsen
**Resumé**: Brugeren er altid informeret om systemets forbindelsesstatus via statusbjælken.

### Fuldstændigt Use Case
**Titel**: Vis datatilslutningsstatus i statusbjælken  
**Omfang**: Slottets Drifttavlen UI  
**Niveau**: Bruger mål  
**Aktører**:  
- Slutbruger  
- System  
**Relaterede krav**:  
- Systemet skal give realtidsfeedback om forbindelsesstatus
- Statusbjælken skal altid være synlig
**Forudsætninger**:  
- Brugeren er logget ind i systemet
- Systemet kører
**Hovedforløb**:  
1. Bruger åbner eller interagerer med systemet  
2. Systemet tjekker den aktuelle datatilslutningsstatus  
3. Statusbjælken viser den aktuelle forbindelsesstatus  
4. Hvis forbindelsestilstanden ændres, opdateres statusbjælken automatisk  
**Udvidelser**:
    a: Når som helst, hvis forbindelsen ikke kan bestemmes, viser statusbjælken en fejl eller ukendt tilstand  
    b: Når som helst, hvis forbindelsen mistes, viser statusbjælken 'offline' og forsøger at genetablere forbindelsen  
        1: Hvis genoprettelsen lykkes, opdateres statusbjælken til 'online'  
        2: Hvis genoprettelsen fejler, forbliver statusbjælken 'offline' og informerer brugeren  
    2a: Hvis systemet ikke kan tjekke forbindelsen, logges en fejl  
    4a: Hvis statusbjælken ikke kan opdatere, informerer systemet brugeren  
**Eftertilstande**:  
- Brugeren er altid klar over den aktuelle forbindelsesstatus
- Systemet logger eventuelle forbindelsesfejl
