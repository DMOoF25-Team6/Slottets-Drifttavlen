## Wireframe

The Dashboard displays resident cards. When an employee selects a resident, the system shows the current notes for that resident and allows adding, editing, and deleting notes.

Overview layout (Dashboard with selected resident):

```text
+----------------------------------------------------------------------------------+
|  SLOTTETS DRIFTTAVLEN                                            [Shift: D/E/N] |
+----------------------------------------------------------------------------------+
|  Residents (cards)                                                               |
|  +----------------+  +----------------+  +----------------+  +----------------+  |
|  | Initials: AB   |  | Initials: CD   |  | Initials: EF   |  | Initials: GH   |  |
|  | Notes:  (2)    |  | Notes:  (0)    |  | Notes:  (1)    |  | Notes:  (0)    |  |
|  +----------------+  +----------------+  +----------------+  +----------------+  |
|                                                                                  |
|  Selected resident details (this wireframe)                                      |
|  +--------------------------------------------------------------------------+    |
|  | Resident: AB (initials only)                                             |    |
|  | Notes                                                                    |    |
|  | [confirmation / error message area]                                      |    |
|  +--------------------------------------------------------------------------+    |
+----------------------------------------------------------------------------------+
```

Detail layout — Selected resident notes:

```text
+----------------------------------------------------------------------------------+
|  Resident: AB (Initials)                                              [Notes]   |
+----------------------------------------------------------------------------------+
|  Current notes                                                                   |
|  +--------------------------------------------------------------------------+    |
|  | Note text                                                     Actions    |    |
|  +--------------------------------------------------------------------------+    |
|  | "Needs extra fluids today" (EditedAt: 09:15 20-04-2026)        [Edit] [Del]   |
|  | "Family visit at 15:00" (EditedAt: 10:02 20-04-2026)           [Edit] [Del]   |
|  +--------------------------------------------------------------------------+    |
|                                                                                  |
|  Add note                                                                        |
|  Note: [                                                                    ]    |
|        [                                                                    ]    |
|        [                                                                    ]    |
|        [Add]                                                                     |
|                                                                                  |
|  Feedback                                                                        |
|  Confirmation: "Note saved" (shown after add/edit/delete)                        |
|  Error:         "Note could not be saved" (shown on save failure)                |
+----------------------------------------------------------------------------------+
```

Edit note state (inline for one row):

```text
+--------------------------------------------------------------------------+
| "[                                                           ]" [Save] [Cancel]|
+--------------------------------------------------------------------------+
```

- Initials are read-only (no full resident names).
- Notes are listed per resident; the list may contain zero or more notes.
- Add/Edit/Delete are the only actions shown for this use case.

## Notes
- This wireframe reflects UC-002: view current notes, add, edit, delete, and show confirmation/error feedback.
- Each note displays the EditedAt timestamp sourced from ResidentNoteDto.
- No additional attributes (e.g., author) are shown in the UI because they are not required by UC-002.
- The placement is shown as an in-page section on the Dashboard to avoid introducing separate screens/modals not specified in UC-002.
