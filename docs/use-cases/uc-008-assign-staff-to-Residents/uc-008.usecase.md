## Use Case Description

## User Story
As an authorized staff member, I want to assign staff to Residents during a shift, so responsibility is clear and care tasks are not missed.

### Brief Use Case
**Title**: Assign Staff to Residents  
**Success Flow**: An authorized user assigns one or more staff members to Residents during a shift, and the system updates the assignments and logs the changes.


### Casual Use Case
**Title**: Assign Staff to Residents  
**Scope**: Staff Assignment  
**Level**: User Goal  
**Actors**:
- Authorized Staff Member
- System
**Main Flow**:
1: Authorized Staff Member opens the staff assignment overview for a shift.
2: System displays Residents and available staff members.
3: Authorized Staff Member assigns one or more staff members to Residents.
4: System validates that each Resident has at least one assigned staff member.
5: System saves the assignments.
6: System updates the assignment overview for all staff members.
7: System logs the assignment changes in the audit trail.
**Main Extensions**:
3a: If user does not have permission, system displays authorization error.
4a: If a Resident has no assigned staff member, system displays validation warning.
5a: If error occurs while saving assignments, system displays error.
6a: If assignment changes during shift, system updates overview and logs changes.
**Summary**: The system provides clear responsibility and visibility of staff assignments for Residents during shifts.


### Fully Dressed Use Case
**Title**: Assign Staff to Residents  
**Scope**: Staff Assignment  
**Level**: User Goal  
**Actors**:
- Authorized Staff Member
- System
**Related Requirements**:
- Assign staff members to Residents during shifts
- Ensure each Resident has at least one assigned staff member
- Allow assignment updates during active shifts
- Display clear responsibility overview for staff members
- Log assignment changes in the audit trail
- Restrict assignment actions to authorized roles

**Preconditions**:
- System is operational
- User is logged into the system
- User has permission to assign staff
- Residents and staff members exist in the system
- A shift is active or scheduled

**Main Flow**:
1: Authorized Staff member opens the staff assignment overview for a shift.
2: System displays Residents and available staff members.
3: Authorized Staff member assigns one or more staff members to Residents.
4: System validates that each Resident has at least one assigned staff member.
5: System saves the assignments.
6: System updates the assignment overview for all staff members.
7: System logs the assignment changes in the audit trail.

**Extensions**:
a: At any time, if an error occurs, system displays error and retry option.
b: At any time, if assignment data cannot be loaded, system displays error message.
3a: If user does not have permission, system displays authorization error.
4a: If a Resident has no assigned staff member, system displays validation warning.
5a: If error occurs while saving assignments, system displays error and retry option.
6a: If assignment changes during shift, system updates overview and logs changes.

**Postconditions**:
- Each Resident has at least one assigned staff member during the shift
- Staff assignments are visible to authorized users
- Assignment changes are stored in the audit trail
- Staff can identify responsibility for Residents during shifts
