## Sequence Diagram: Assign Staff to Residents

### Presentation → Application
```mermaid
sequenceDiagram
    actor AuthorizedStaffMember
    participant StaffAssignmentPage as Presentation Layer
    participant StaffAssignmentController as API/Controller Layer
    participant StaffAssignmentService as Application Layer

    AuthorizedStaffMember ->> StaffAssignmentPage: SaveStaffAssignment(resident, staffMember, shift)
    StaffAssignmentPage ->> StaffAssignmentController: PostStaffAssignment(staffAssignmentDto)
    StaffAssignmentController ->> StaffAssignmentService: AssignStaffToResident(staffAssignmentDto)

    alt UserNotAuthorized
        StaffAssignmentService -->> StaffAssignmentController: Error(authorizationError)
        StaffAssignmentController -->> StaffAssignmentPage: Error(message)
    else ResidentHasNoAssignedStaff
        StaffAssignmentService -->> StaffAssignmentController: ValidationWarning()
        StaffAssignmentController -->> StaffAssignmentPage: ValidationWarning()
    else Success
        StaffAssignmentService -->> StaffAssignmentController: StaffAssignmentDto
        StaffAssignmentController -->> StaffAssignmentPage: UpdatedAssignmentOverviewDto
        StaffAssignmentPage -->> AuthorizedStaffMember: DisplayConfirmation()
    end
```

### Application → Infrastructure
```mermaid
sequenceDiagram
    participant StaffAssignmentService as Application Layer
    participant StaffAssignmentRepository as Repository Layer
    participant Database as Database

    StaffAssignmentService ->> StaffAssignmentRepository: SaveStaffAssignment(staffAssignment)
    StaffAssignmentRepository ->> Database: InsertOrUpdateStaffAssignment(staffAssignment)
    Database -->> StaffAssignmentRepository: SaveResult
    StaffAssignmentRepository -->> StaffAssignmentService: StaffAssignment

    StaffAssignmentService ->> StaffAssignmentRepository: GetUpdatedAssignmentOverview(shift)
    StaffAssignmentRepository ->> Database: FetchAssignmentsByShift(shift)
    Database -->> StaffAssignmentRepository: AssignmentData
    StaffAssignmentRepository -->> StaffAssignmentService: UpdatedAssignmentOverview
```

## Notes
- DTOs are used for data transfer between the UI, controller, and service.
- The service validates authorization and assignment rules before saving.
- The repository handles persistence of StaffAssignment data.
- The database stores the staff assignment for the selected Resident and Shift.
- Assignment changes must be logged according to the audit trail requirement.
