## System Sequence Diagram

---

```mermaid
sequenceDiagram
    actor AuthorizedStaffMember
    participant System

    AuthorizedStaffMember ->> System: openStaffAssignmentOverview(shift)

    alt userNotAuthorized
        System -->> AuthorizedStaffMember: displayAuthorizationError()

    else errorLoadingAssignmentData
        System -->> AuthorizedStaffMember: displayErrorMessage()

    else success
        System -->> AuthorizedStaffMember: displayResidentsAndAvailableStaff(residents, staffMembers)
    end

    AuthorizedStaffMember ->> System: assignStaffToResident(resident, staffMember, shift)

    alt residentHasNoAssignedStaff
        System -->> AuthorizedStaffMember: displayValidationWarning()

    else errorSavingAssignment
        System -->> AuthorizedStaffMember: displayErrorAndRetryOption()

    else success
        System -->> AuthorizedStaffMember: displayUpdatedAssignmentOverview()
    end
```

## Notes
- The system validates that the Resident has at least one assigned staff member.
- Only authorized roles can assign or update staff assignments.
- Assignment changes are logged in the audit trail.
- The updated assignment overview shows responsibility during the shift.
