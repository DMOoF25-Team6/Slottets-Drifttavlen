## Use Case Description

## User story
As a user, I want a navigation bar in the main layout so that I can easily access all major sections of the system from anywhere in the application.

### Brief Use Case
**Title**: Add navigation bar to main layout  
**Success Flow**:  
The user opens the application and sees a navigation bar at the top of the main layout. The navigation bar provides links to all major sections (Resident Overview, Tasks, Medicine Status, Administration, etc.). The user clicks a link and is taken to the selected section. The navigation bar adapts to the user's role, showing only the relevant links.

### CaseCasual Use Case
**Title**: Add navigation bar to main layout  
**Scope**: Application UI  
**Level**: User goal  
**Actors**:  
- User (Staff, Admin, Leader)
**Main Flow**:  
1: User logs in and accesses the main layout.
2: User sees the navigation bar with links to major sections.
3: User clicks a link to navigate to a section.
**Main Extensions**:  
    1a: User is not logged in – navigation bar shows only login/register links.  
    3a: User clicks a link for which they lack permission – system displays an access denied message.  
**Summary**: Users can efficiently navigate between all major sections of the system using a consistent navigation bar.

### Fully Dressed Use Case
**Title**: Add navigation bar to main layout  
**Scope**: Application UI  
**Level**: User goal  
**Actors**:  
- User (Staff, Admin, Leader)  
**Related Requirements**:  
- REQ-U-003  
- docs/milestones-gateways.md (Gateway 10: Status Bar)  
**Preconditions**:  
- User is authenticated.
- Application main layout is loaded.
**Main Flow**:  
1. User accesses the main layout after login.  
2. System displays a navigation bar with links to all major sections.  
3. User clicks a navigation link.  
4. System navigates to the selected section and updates the view.  
5. Navigation bar adapts to user claims, showing only permitted links.  
**Extensions**:
    a: At any time, user logs out – back to login page.
    b: At any time, user session expires – back to login page.  
    3a: User clicks a link for which they lack permission – system displays an access denied message.  
**Postconditions**:  
- User is able to access all permitted sections from the navigation bar.
- Navigation bar reflects the user's current authentication and authorization state.
