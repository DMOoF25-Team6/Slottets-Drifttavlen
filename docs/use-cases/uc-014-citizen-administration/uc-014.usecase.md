## Use Case Description

## User story
As an administrator, I want to create a new citizen record in the system, so that the citizen can be managed and assigned to staff and tasks.

### Brief Use Case
**Title**: Create Citizen (Administration)  
**Success Flow**:  
The administrator selects the option to create a new citizen, enters the required citizen information, and confirms the creation. The system validates the input, creates the citizen record, and makes it available for assignment and management.

### Casual Use Case
**Title**: Create Citizen (Administration)  
**Scope**: Slottets Drifttavlen (Administration)  
**Level**: User goal  
**Actors**:  
- Administrator

**Main Flow**:  
1: Administrator navigates to the citizen administration section.  
2: Administrator selects 'Create Citizen'.  
3: System displays the citizen creation form.  
4: Administrator enters required citizen information (e.g., name, date of birth, status, etc.).  
5: Administrator submits the form.  
6: System validates the input.  
7: System creates a new citizen record.  
8: System confirms creation and displays the new citizen in the list.

**Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;4a: Required information is missing or invalid.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: System highlights missing/invalid fields and prompts for correction.  
&nbsp;&nbsp;&nbsp;&nbsp;6a: System validation fails (e.g., duplicate citizen).  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: System displays an error and does not create the citizen.

**Summary**: The administrator can create a new citizen by entering required information. The system validates and saves the new citizen, or prompts for corrections if needed.

### Fully Dressed Use Case
**Title**: Create Citizen (Administration)
**Primary Actor**: Administrator
**Scope**: Slottets Drifttavlen (Administration)
**Level**: User goal
**Stakeholders and Interests**:
- Administrator: Wants to efficiently add new citizens to the system.
- Staff: Needs new citizens to be available for assignment and management.
- System: Ensures data integrity and prevents duplicates.

**Preconditions**:

**Main Success Scenario**:
1. Administrator navigates to the citizen administration section.
2. Administrator selects 'Create Citizen'.
3. System displays the citizen creation form.
4. Administrator enters all required citizen information.
5. Administrator submits the form.
6. System validates the input.
7. System creates a new citizen record.
8. System confirms creation and displays the new citizen in the list.

**Extensions**:

**Postconditions**:
- New citizen is available in the system for assignment and management.
- Audit trail records the creation event.
