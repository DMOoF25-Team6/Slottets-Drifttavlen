## Use Case Description

## User story
As an administrator, I want to create a new Resident record in the system, so that the Resident can be managed and assigned to staff and tasks.

### Brief Use Case
**Title**: Create Resident (Administration)  
**Success Flow**:  
The administrator selects the option to create a new Resident, enters the required Resident information, and confirms the creation. The system validates the input, creates the Resident record, and makes it available for assignment and management.

### Casual Use Case
**Title**: Create Resident (Administration)  
**Scope**: Slottets Drifttavlen (Administration)  
**Level**: User goal  
**Actors**:  
- Administrator

**Main Flow**:  
1: Administrator navigates to the Resident administration section.  
2: Administrator selects 'Create Resident'.  
3: System displays the Resident creation form.  
4: Administrator enters required Resident information (e.g., name, date of birth, status, etc.).  
5: Administrator submits the form.  
6: System validates the input.  
7: System creates a new Resident record.  
8: System confirms creation and displays the new Resident in the list.

**Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;4a: Required information is missing or invalid.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: System highlights missing/invalid fields and prompts for correction.  
&nbsp;&nbsp;&nbsp;&nbsp;6a: System validation fails (e.g., duplicate Resident).  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: System displays an error and does not create the Resident.

**Summary**: The administrator can create a new Resident by entering required information. The system validates and saves the new Resident, or prompts for corrections if needed.

### Fully Dressed Use Case
**Title**: Create Resident (Administration)
**Primary Actor**: Administrator
**Scope**: Slottets Drifttavlen (Administration)
**Level**: User goal
**Stakeholders and Interests**:
- Administrator: Wants to efficiently add new Residents to the system.
- Staff: Needs new Residents to be available for assignment and management.
- System: Ensures data integrity and prevents duplicates.

**Preconditions**:
- Administrator is authenticated and authorized.
- Resident administration logic is available.

**Main Success Scenario**:
1. Administrator navigates to the Resident administration section.
2. Administrator selects 'Create Resident'.
3. System displays the Resident creation form.
4. Administrator enters all required Resident information.
5. Administrator submits the form.
6. System validates the input.
7. System creates a new Resident record.
8. System confirms creation and displays the new Resident in the list.

**Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;4a. Required information is missing or invalid:  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System highlights missing/invalid fields and prompts for correction.  
&nbsp;&nbsp;&nbsp;&nbsp;6a. System validation fails (e.g., duplicate Resident):  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays an error and does not create the Resident.  

**Postconditions**:
- New Resident is available in the system for assignment and management.
- Audit trail records the creation event.
