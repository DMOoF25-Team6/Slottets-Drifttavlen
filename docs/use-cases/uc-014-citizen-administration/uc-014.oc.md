## Operation Contract

### navigateToResidentAdministration
- **Preconditions**: Administrator is authenticated and authorized; Resident administration logic is available.
- **Postconditions**: Resident administration section is displayed to the administrator.

### selectCreateResident
- **Preconditions**: Resident administration section is displayed.
- **Postconditions**: Resident creation form is displayed to the administrator.

### submitResidentForm
- **Preconditions**: Resident creation form is displayed; administrator has entered Resident information.
- **Postconditions**: 
    - If information is missing or invalid: System highlights missing/invalid fields and prompts for correction.
    - If duplicate Resident: System displays an error and does not create the Resident.
    - If valid input: System creates a new Resident record, confirms creation, and displays the new Resident in the list. Audit trail records the creation event.
