## Wireframe

UC-009 provides a read-only view for:
- Viewing previous handovers/overlaps by date and shift.
- Viewing traceability (audit trail) for changes: who changed what and when.

Overview layout (Dashboard entry point):

```text
+----------------------------------------------------------------------------------+
|  SLOTTETS DRIFTTAVLEN                                            [Shift: D/E/N] |
+----------------------------------------------------------------------------------+
|  [Main dashboard content — resident cards, shared sections, etc.]                |
|                                                                                  |
|  Actions:                                                   [History & trace]    |
+----------------------------------------------------------------------------------+
```

Detail layout — History & Traceability view:

```text
+----------------------------------------------------------------------------------+
|  HISTORY & TRACEABILITY                                        [Back]            |
+----------------------------------------------------------------------------------+
|  Select overlap (handover)                                                        |
|  From: [YYYY-MM-DD]  To: [YYYY-MM-DD]  Shift: [D/E/N/All]   [Apply] [Clear]      |
|                                                                                  |
|  Overlaps (read-only list)                                                       |
|  +----------------------------------------------------------------------------+  |
|  | Date       | Shift | Department | Summary                     | [Open]     |  |
|  +----------------------------------------------------------------------------+  |
|  | 2026-05-07 | E     | Slottet    | 12 residents, 4 tasks, 3 notes | [Open]  |  |
|  | 2026-05-07 | D     | Slottet    | 12 residents, 5 tasks, 1 note  | [Open]  |  |
|  | 2026-05-06 | N     | Slottet    | 12 residents, 3 tasks, 2 notes | [Open]  |  |
|  +----------------------------------------------------------------------------+  |
|                                                                                  |
|  Selected overlap (read-only snapshot)                                           |
|  +----------------------------------------------------------------------------+  |
|  | Overlap: 2026-05-07 / Evening / Slottet                                      |  |
|  |                                                                            |  |
|  |  [Read-only snapshot of the handover layout (same sections as Dashboard)]  |  |
|  |  - Residents (initials + status)                                            |  |
|  |  - Notes/messages                                                           |  |
|  |  - Tasks                                                                    |  |
|  |  - Shared sections (e.g., PhoneList)                                         |  |
|  +----------------------------------------------------------------------------+  |
|                                                                                  |
|  Traceability (audit trail)                                                      |
|  Filters: Entity: [All ▼]  Change: [All ▼]  Search: [              ]  [Filter]   |
|                                                                                  |
|  +----------------------------------------------------------------------------+  |
|  | Timestamp        | Entity          | Change  | Changed by      | Description |  |
|  +----------------------------------------------------------------------------+  |
|  | 2026-05-07 10:02 | ResidentNote    | Updated | user@slottet.dk  | Edited note |  |
|  | 2026-05-07 09:15 | PhoneAssignment | Updated | user@slottet.dk  | Assigned ph |  |
|  | 2026-05-06 22:01 | Task            | Created | user@slottet.dk  | Added task  |  |
|  +----------------------------------------------------------------------------+  |
|                                                                                  |
|  Selected audit entry (read-only details)                                        |
|  +----------------------------------------------------------------------------+  |
|  | Timestamp:   2026-05-07 10:02                                                |  |
|  | Entity:      ResidentNote                                                    |  |
|  | Change type: Updated                                                         |  |
|  | Changed by:  user@slottet.dk                                                 |  |
|  | Description: [Edited note: "Needs extra fluids today"]                      |  |
|  +----------------------------------------------------------------------------+  |
+----------------------------------------------------------------------------------+
```

- All history and audit data is read-only in this use case.
- “[Open]” shows the selected overlap snapshot in the “Selected overlap” area (no separate modal/page).
- Audit trail rows provide: timestamp, what changed (entity), change type, who changed it, and a short description.

## Notes
- This wireframe supports REQ-F-006 (history + traceability / audit trail).
- The overlap list is shown as a separate read-only list to avoid introducing editing actions in UC-009.
- The “Changed by” value is shown as a user identifier (e.g., email/username) to keep it unambiguous.
- Exact overlap summary values are illustrative; the intent is to provide an at-a-glance overview per shift.
